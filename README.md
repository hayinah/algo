# Algorithm Analysis Project

## Overview
This project focuses on analyzing 8 anonymous algorithms provided in pseudocode. The goal is to determine what each algorithm does, analyze its time complexity (best and worst case), implement it in C++, and compare theoretical performance with actual runtime results.

## Objectives
- Identify the purpose of each algorithm
- Analyze time complexity (Big-O) for best and worst cases
- Implement each algorithm in C++
- Test performance using different input sizes
- Compare predicted vs measured runtime
- Explain any differences observed

## Technologies Used
- C++
- Standard Template Library (STL)
- <chrono> library for runtime measurement

## Project Structure
- algorithm1.cpp – Subset generation (backtracking)
- algorithm2.cpp – Bucket/Radix-style sorting
- algorithm3.cpp – Longest Increasing Subsequence
- algorithm4.cpp – Quickselect (kth smallest element)
- algorithm5.cpp – Merge Sort (bottom-up)
- algorithm6.cpp – Permutation-based sorting check
- algorithm7.cpp – Counting Sort
- algorithm8.cpp – Cocktail Shaker Sort
- results/ – Runtime data and analysis
- report/ – Final written report

## Testing Approach
- Input sizes vary based on algorithm complexity
- Polynomial algorithms: N = 50 to 500
- Exponential algorithms: smaller N (5–12)
- Factorial algorithms: very small N (3–8)
- Each test is run multiple times and averaged

## Status
In progress – algorithms are being analyzed, implemented, and tested.
